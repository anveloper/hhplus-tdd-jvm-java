## TDD로 개발하기

### 기본과제

- API 요구사항

  - [ ] PATCH `/point/{id}/charge` : 포인트를 충전한다.
  - [ ] PATCH `/point/{id}/use` : 포인트를 사용한다.
  - [ ] GET `/point/{id}` : 포인트를 조회한다.

- 기능 요구사항

  - [ ] GET `/point/{id}/histories/` : 포인트 내역을 조회한다.
  - [ ] 잔고가 부족할 경우, 초인트 사용은 실패해야한다.

### 심화과제

- [ ] 같은 사용자가 동시에 충전할 경우, 해당 요청 모두 정상적으로 반영되어야 한다.
